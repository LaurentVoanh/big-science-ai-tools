 INSTRUCTIONS POUR UN SYSTÈME D'AUTO-RECHERCHE AVEC AUTO-RENFORCEMENT


🎯 OBJECTIFS PRINCIPAUX


Auto-recherche intelligente :

L’IA choisit, exécute, et optimise les requêtes sans intervention humaine.
Auto-renforcement : Elle apprend de chaque session pour améliorer les suivantes.


Communication sympathique :

L’IA parle à l’utilisateur à chaque étape (ex: "Je vais maintenant interroger PubMed pour trouver des études sur BRCA1…").
Ton amical et pédagogique (ex: "Super ! J’ai trouvé 45 études sur PubMed. Je passe à ClinVar…").


Auto-documentation :

L’IA rédige des articles sur son propre fonctionnement (ex: "Comment j’ai optimisé ma recherche sur BRCA1").
Elle explique ses améliorations (ex: "J’ai appris que pour les gènes, UniProt est plus fiable que Wikipedia").


Gestion des prompts :

Les prompts IA sont stockés dans des fichiers texte (ex: prompts/selection_sujet.txt).
L’IA crée de nouveaux prompts pour tester des stratégies (ex: prompts/nouvelle_strategie_brca1.txt).
Elle teste ses propres prompts pour s’améliorer.


Intégration des mini-apps dans les articles :

Les mini-apps générées (PHP/SQLite) sont intégrées directement dans les articles (ex: "Pour tester cette hypothèse, utilisez l’application ci-dessous : [Lien vers la mini-app]").
Les résultats des mini-apps sont affichés dans l’article (ex: "Les tests montrent que 80% des patientes avec la mutation c.5266dupC résistent à Olaparib (p-value = 0.001)").


Communication contextuelle :

L’utilisateur peut choisir le contexte (ex: "Focus sur l’oncologie", "Évite les sources peu fiables").
L’IA adapte ses prompts en fonction du contexte.

📌 RÈGLES POUR LES PROMPTS (FICHIERS TEXTE)


Stockage :

Tous les prompts sont stockés dans prompts/ (ex: prompts/selection_sujet.txt).
Chaque fichier contient un seul prompt, avec :

Contexte (ex: rôle de l’IA, données disponibles).
Mission (ce que l’IA doit faire).
Format de sortie (JSON, Markdown, etc.).



Création de nouveaux prompts :

L’IA crée de nouveaux fichiers dans prompts/ pour tester des stratégies innovantes.
Exemple : prompts/nouvelle_strategie_brca1.txt (pour une stratégie spécifique à BRCA1).


Test des prompts :

L’IA teste ses propres prompts en les exécutant et en évaluant les résultats.
Si un prompt donne de bons résultats, il est conservé et réutilisé.
Si un prompt donne de mauvais résultats, il est modifié ou supprimé.


Versioning :

Les prompts sont versionnés (ex: prompts/selection_sujet_v1.txt, prompts/selection_sujet_v2.txt).
L’IA stocke les performances de chaque version (ex: success_rate, note_utilisateur).

🔄 LOGIQUE D’AUTO-RENFORCEMENT
1. Mise à Jour des Stratégies (query_strategies)

Si une requête réussit (hits > 0) :

success_rate += 0.1
times_used += 1
Log : "Stratégie optimisée pour [source] avec le terme: [term]"

Si une requête échoue (hits = 0) :

success_rate -= 0.2
Proposition d’un terme alternatif (via un nouveau prompt).
Log : "Échec pour [source] avec le terme: [term]. Nouvelle tentative avec: [new_term]"

2. Stockage des Améliorations (improvements)

Après chaque session, l’IA :

Identifie les points forts (ex: "Découvert que UniProt > ClinVar pour les gènes").
Identifie les axes d’amélioration (ex: "Ajouter la source gnomAD pour les mutations rares").
Propose une action concrète (ex: "Prioriser UniProt pour les sujets génétiques").
Stocke tout dans improvements pour les prochaines sessions.

3. Création de Nouveaux Prompts

Si l’IA découvre une nouvelle stratégie efficace, elle :

Crée un nouveau fichier dans prompts/ (ex: prompts/strategie_brca1.txt).
Teste le nouveau prompt sur plusieurs sessions.
Si le prompt est efficace (success_rate > 0.8), elle le conserve.
Si le prompt est inefficace, elle le modifie ou supprime.

4. Auto-Documentation

L’IA rédige des articles sur son évolution (ex: "Comment j’ai optimisé ma recherche sur BRCA1").
Ces articles sont stockés dans articles/system/ et mis à jour régulièrement.
